US003 — Login de Usuário · 5 pts · Alta · Depende de US001

"Como usuário interno, quero fazer login com e-mail e senha, para acessar a área restrita correspondente ao meu perfil."
Backend
US003-BE-01
Endpoint POST /auth/login
Dev 4Alta
Checklist
Receber e-mail + senha, verificar com bcrypt
Credenciais inválidas → mensagem genérica
Após 5 falhas consecutivas → bloquear conta por 15 min
Sucesso → retornar JWT com payload: user_id, perfil, exp
Expiração por inatividade: 8 horas (SESSION_TIMEOUT_HOURS=8)
Log de auditoria: logar tentativas falhas e logins bem-sucedidos
US003-BE-02
Middleware de autenticação JWT
Dev 4Alta
Checklist
Validar token em rotas protegidas
Extrair perfil do token e disponibilizar no contexto
Token expirado → 401 com mensagem clara
Token inválido → 401
Frontend
US003-FE-01
Tela de login
Dev 4Alta
Checklist
Campos: e-mail e senha
Erro genérico para credenciais inválidas
Link "Esqueci minha senha" → rota de reset (US004)
Armazenar JWT via HttpOnly cookie (mais seguro que localStorage)
US003-FE-02
Redirect por perfil + Auth Guards
Dev 4Alta
Checklist
Login COORDENADOR → /coordenador/dashboard
Login ALUNO → /aluno/dashboard
Login ORIENTADOR → /orientador/dashboard
Guard Angular protegendo rotas de cada perfil
Exibir nome e perfil do usuário em todas as páginas restritas
Sessão expirada → redirecionar para login com mensagem
Testes

Casos de teste — US003

US004 — Redefinição de Senha · 3 pts · Média · Depende de US001

"Como usuário, quero solicitar a redefinição de senha por e-mail, para recuperar acesso à minha conta caso esqueça minha senha."
Backend
US004-BE-01
Endpoint POST /auth/solicitar-reset
Dev 2Média
Checklist
Receber e-mail; sempre retornar mensagem genérica
Gerar token único (uuid4) com TTL de 2 horas
Salvar token: user_id, token, expira_em, usado
Enviar e-mail com link: /reset-senha?token=xxx
US004-BE-02
Endpoint POST /auth/confirmar-reset
Dev 2Média
Checklist
Receber token + nova senha
Validar: existe, não expirou, não foi usado
Token inválido/expirado → 400 com mensagem clara
Senha nova mínimo 8 caracteres + hash bcrypt
Marcar token como usado=true após uso
Log de auditoria: RESET_SENHA, user_id, timestamp
Frontend
US004-FE-01
Tela de solicitação de reset
Dev 3Média
Checklist
Campo e-mail + botão enviar
Após envio: "Se o e-mail estiver cadastrado, você receberá as instruções"
Link de volta para o login
US004-FE-02
Tela de nova senha (via link do e-mail)
Dev 3Média
Checklist
Ler token da query string (?token=xxx)
Campos: nova senha + confirmação
Validação: senhas coincidem + mínimo 8 caracteres
Token inválido/expirado → mensagem + link para novo reset
Sucesso → redirecionar para login com confirmação
Testes
US004-QA-01
Casos de teste — US004
Tester 2
Cenários obrigatórios
E-mail válido → resposta genérica, e-mail disparado, token no banco
E-mail inexistente → mesma resposta genérica
Token válido + nova senha OK → senha alterada, token usado
Mesmo token usado 2x → erro 400
Token expirado (>2h) → erro 400
Token inválido/adulterado → erro 400
Nova senha < 8 caracteres → erro de validação
Após reset: senha antiga falha, nova senha funciona
